1.1 What is ACME DataBridge?
ACME DataBridge is an enterprise data integration and transformation platform that enables organizations to move, transform, and orchestrate data across cloud, on-premise, and hybrid environments. DataBridge supports real-time streaming, batch processing, and event-driven architectures, with a no-code visual designer for citizen integrators and a full SDK for developers.

1.2 Core Capabilities
- Connectors: 200+ pre-built connectors for SaaS apps (Salesforce, SAP, NetSuite), databases (PostgreSQL, Oracle, Snowflake), message queues (Kafka, RabbitMQ), and file systems (S3, SFTP, SharePoint)
- Transformations: Visual mapping editor, SQL transformations, Python/JavaScript custom functions, AI-assisted data mapping (Beta)
- Orchestration: Workflow scheduler, dependency management, retry logic with exponential backoff
- Monitoring: Real-time pipeline health dashboard, data quality scoring, SLA alerting
- Security: End-to-end encryption, field-level masking, role-based access control (RBAC), SOC 2 Type II certified

1.3 Deployment Options
- Cloud SaaS: ACME-hosted on AWS us-east-1 and eu-west-1 (GDPR-compliant). 99.9% SLA for Standard, 99.95% for Enterprise.
- Private Cloud: Deploy DataBridge in your own AWS, Azure, or GCP account using our Terraform modules. Minimum: 4 vCPU, 16 GB RAM per node.
- On-Premise: Available for Enterprise tier. Requires Linux (RHEL 8+ or Ubuntu 20.04+), minimum 8 vCPU, 32 GB RAM, 500 GB storage. Air-gapped environments supported.

2.1 Account Setup
New accounts are provisioned within 30 minutes of signup confirmation. Upon first login:
1. Set up Multi-Factor Authentication (MFA) — required for all accounts
2. Create your first Workspace (a logical container for pipelines, connections, and team members)
3. Invite team members (Workspace Admin can manage user roles)
4. Configure your first connection using the Connection Wizard

2.2 User Roles and Permissions
DataBridge uses Role-Based Access Control (RBAC) at both the organization and workspace level:
- Organization Admin: Full control; manages billing, SSO, and creates/deletes workspaces
- Workspace Admin: Manages users, connections, and all pipelines within a workspace
- Pipeline Editor: Creates and modifies pipelines; cannot manage connections or users
- Viewer: Read-only access to pipeline runs, logs, and monitoring dashboards
- API User: Service account role for programmatic access only; no UI access

Custom roles are available on Enterprise tier. Contact your Customer Success Manager to configure.

2.3 Pricing Tiers
- Starter: $299/month — up to 5 pipelines, 10M records/month, 2 workspaces, email support
- Growth: $999/month — up to 25 pipelines, 100M records/month, 10 workspaces, Slack support
- Enterprise: Custom pricing — unlimited pipelines, custom record limits, unlimited workspaces, dedicated CSM, SLA guarantees, SSO/SAML, on-premise option

All tiers include a 14-day free trial. No credit card required for trial. Usage above the record limit is billed at $0.10 per million additional records (Starter) or $0.06 per million (Growth).

CHAPTER 3: BUILDING PIPELINES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3.1 Pipeline Components
A DataBridge pipeline consists of:
- Source: Where data originates. Supported source types: polling (scheduled fetch), webhook (push), CDC (Change Data Capture), file drop (S3/SFTP watch), streaming (Kafka/Kinesis).
- Transformations: Zero or more transformation steps applied in sequence. Steps can be: field mapping, filter, aggregate, join (cross-source), enrich (API lookup), deduplicate, split, and merge.
- Destination: Where data lands. Supports: databases, data warehouses, SaaS apps, message queues, files, and webhooks.

3.2 Trigger Types
- Schedule: Run at fixed intervals (cron syntax supported) or natural language ("every weekday at 6 AM EST")
- Event-Driven: Trigger when a new record is created/updated in a connected source (requires CDC or webhook support)
- API-Triggered: Start a pipeline run via REST API call (use for custom orchestration)
- Manual: Run on-demand from the UI or CLI

3.3 Error Handling and Retries
Failed pipeline runs are retried automatically based on the configured retry policy:
- Default: 3 retries with exponential backoff (delays: 1 min, 4 min, 16 min)
- Maximum retries: 10 (configurable per pipeline)
- Dead-letter queue: Records that exceed max retries are routed to a DLQ in S3 or sent as an alert

Error notifications can be sent to: email, Slack, PagerDuty, or webhook. Configure under Pipeline > Settings > Alerts.

3.4 Data Quality Rules
DataBridge includes a built-in Data Quality engine. Rules can be applied at any pipeline stage:
- Schema validation: Enforce data types, required fields, allowed values
- Statistical checks: Flag anomalies (e.g., daily record count drops >30% vs. 7-day average)
- Custom SQL assertions: Write SQL WHERE clauses that records must satisfy
- Cross-pipeline consistency: Compare aggregate values across pipelines

Quality failures can be configured to: warn only, quarantine failing records, or halt the pipeline.

CHAPTER 4: API REFERENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4.1 Authentication
All API calls require an API key passed in the Authorization header: "Authorization: Bearer {API_KEY}". API keys are created per workspace under Settings > API Keys. Keys can be scoped to specific permissions (read-only, run pipelines, full admin). Keys expire after 365 days unless set to never-expire (Enterprise only). Rotate keys promptly if compromised.

4.2 Base URL
Production: https://api.databridge.acmecorp.com/v3/
EU Region: https://api.eu.databridge.acmecorp.com/v3/
Sandbox: https://api.sandbox.databridge.acmecorp.com/v3/ (uses production schema, no billing)

4.3 Key Endpoints
- GET /workspaces — List all workspaces accessible to the authenticated user
- GET /workspaces/{id}/pipelines — List all pipelines in a workspace
- POST /pipelines/{id}/runs — Trigger a pipeline run
- GET /pipelines/{id}/runs/{run_id} — Get run status and metrics
- GET /pipelines/{id}/runs/{run_id}/logs — Retrieve run logs (paginated)
- POST /connections — Create a new connection
- GET /connections/{id}/test — Test connection health
- GET /records/dlq — Retrieve dead-letter queue records (paginated)

4.4 Rate Limits
- Standard: 100 requests/minute per API key
- Enterprise: 1,000 requests/minute per API key
Rate limit headers: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset
HTTP 429 is returned when the limit is exceeded. Implement exponential backoff in client code.

4.5 Webhooks (Outbound)
DataBridge can POST event notifications to your endpoint. Events include: pipeline.run.started, pipeline.run.completed, pipeline.run.failed, data_quality.alert. Webhook payloads are signed with HMAC-SHA256 using your webhook secret. Verify the X-DataBridge-Signature header to authenticate. Retry policy: 5 retries over 24 hours.

CHAPTER 5: INTEGRATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

5.1 Salesforce Integration
The Salesforce connector supports both REST API and Bulk API 2.0. For real-time sync of individual records, use REST. For initial loads or large-volume syncs (>10,000 records), use Bulk API for better performance. Required permissions: API Enabled, View All Data (source), Modify All Data (destination). CDC via Salesforce Platform Events is supported for real-time change capture.

5.2 Snowflake Integration
DataBridge uses Snowflake's native connector for optimized performance. Recommended setup: create a dedicated service account with a dedicated warehouse (size: X-Small for most workloads). Enable key pair authentication for enhanced security. DataBridge supports Snowpipe for streaming inserts and COPY INTO for batch loads. Use the MERGE strategy for upserts to avoid duplicate rows.

5.3 Kafka Integration
For Kafka sources, DataBridge runs a consumer group. Configure the consumer group ID as "databridge-{workspace_id}" to avoid conflicts. Supported authentication: SASL/PLAIN, SASL/SCRAM-SHA-256, mutual TLS. Schema Registry (Confluent) is supported for Avro/Protobuf decoding. Set auto.offset.reset=earliest for initial loads, latest for ongoing sync.

5.4 Microsoft SharePoint / Teams
DataBridge connects to SharePoint Online and OneDrive via Microsoft Graph API. Required permissions (app registration): Files.Read.All or Files.ReadWrite.All, Sites.Read.All. Grant admin consent in Azure AD. File watch mode supports .xlsx, .csv, .json, .xml, and .parquet. SharePoint lists can be synced to any destination.

6.1 Common Error Codes
- DB_CONN_001: Cannot connect to database. Check host, port, and firewall rules. ACME's IP ranges that need to be whitelisted: 52.10.12.0/24, 18.195.30.0/24 (EU).
- SCHEMA_MISMATCH_002: Source schema changed. Open the pipeline mapper to accept schema changes.
- RATE_LIMIT_003: Downstream API rate limit hit. Enable rate limiting in pipeline settings.
- AUTH_EXPIRED_004: Credentials for a connection have expired. Reconnect the affected connection.
- DLQ_FULL_005: Dead-letter queue has reached its limit (default: 100,000 records). Resolve DLQ items before the pipeline can run again.

6.2 Performance Tuning
For pipelines processing >1M records per run:
- Enable parallel processing (up to 8 workers per pipeline on Growth tier, 32 on Enterprise)
- Use bulk-mode connectors where available (Salesforce, Snowflake, S3)
- Avoid row-by-row enrichments; batch API lookups in groups of 100-500
- Partition large datasets by date or ID range to enable restartable runs

6.3 Support and SLA
- Starter: Email support, 24-hour response time for P3/P4, 4 hours for P2, no P1 guarantee
- Growth: Email + Slack support, 8-hour P3/P4, 2-hour P2, 4-hour P1
- Enterprise: 24/7 phone and Slack support, 1-hour P1 response, dedicated Customer Success Manager, quarterly business reviews

Submit support tickets at support.databridge.acmecorp.com. Include your workspace ID, pipeline ID, and run ID in all tickets for faster resolution.
